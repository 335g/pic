# CLIからR2へ直接アップロードし、Backendは署名付きURLを発行する

写真・動画ファイルのアップロード経路として、CLIがCloudflare R2に直接PUTし、Backend（API）はアップロード完了通知を受けてメタデータをD1に記録する。読み出し時はBackendが署名付きURLを発行し、Frontendはそれを経由してR2から直接ダウンロードする。

## 理由

- **Cloudflare Workers Free プランのリクエスト本文上限 (100MB) を回避する**。動画ファイルはこれを超える可能性が高く、Backend経由ではアップロードできない。
- **Backendに大きなバイナリ転送の負荷がかからない**。CLI→R2間は直接PUTされるため、WorkersのCPU時間やメモリを消費しない。
- **CLIがサムネイル生成を担当するアーキテクチャと整合する**。CLIが元ファイルとサムネイルの両方を知っているため、同一のアップロード処理で両方をR2に送れる。

## 考慮した選択肢

- **CLI → Backend → R2（完全経由）**: リクエスト本文100MB制限に引っかかるリスクがあり、特に動画ファイルで問題になる。またWorkersのCPU時間を消費する。
- **Backendがサムネイル生成 + R2保存まで担当**: WorkersのCPU時間制限（Free: 10ms）とWASMバンドルサイズ制限（Free: 3MB）により、実用的な画像処理が難しい。
